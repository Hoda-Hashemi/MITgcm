# Williamson TC1 on a Cubed-Sphere Grid

This experiment adapts MITgcm's `advect_cs` verification case to the
Williamson test case 1 passive-tracer problem.

The case uses:

- a 32 x 32 x 6 cubed-sphere grid;
- one passive salt tracer containing a cosine bell;
- a prescribed solid-body flow with a 12-day period;
- oceanic settings with zero tracer buoyancy;
- daily state and diagnostic output.

All TC1 binary inputs are generated by `input/prepare_run`. The default
rotation-axis angle is `pi/2 - 0.05`:

```sh
cd input
./prepare_run
```

For exact pole crossing:

```sh
./prepare_run 1.5707963267948966
```

The generated files are:

- `grid_cs32.face001.bin` through `grid_cs32.face006.bin`: local copies of
  the standard MITgcm cs-32 grid files;
- `S_init.bin`: cosine-bell passive tracer;
- `U_init.bin`: cubed-sphere C-grid zonal component;
- `V_init.bin`: cubed-sphere C-grid meridional component.

The grid source is discovered from the surrounding MITgcm `verification`
tree. Real files are copied instead of symlinks, so this works afterward:

```sh
cd ../run
cp ../input/* .
```

The velocity is derived from the analytic streamfunction at grid corners.
This makes its finite-volume divergence vanish discretely apart from
roundoff and cubed-sphere edge exchange.

Build with either `code/SIZE.h` for one process or `code/SIZE.h_mpi` for
six MPI processes.
